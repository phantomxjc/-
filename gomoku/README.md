# 五子棋游戏 (Gomoku)

基于 Flask + Canvas 的五子棋网页游戏，支持人机对战和真人对战。

## 项目结构

```
gomoku/
├── app.py           # Flask 后端入口
├── game_logic.py    # 核心游戏逻辑（棋盘、胜负判断）
├── ai_engine.py     # AI 引擎（三难度）
├── templates/
│   └── index.html   # 前端单页面
├── requirements.txt
└── README.md
```

## 功能说明

| 功能 | 说明 |
|------|------|
| 游戏模式 | **人机对战**（PvC） / **真人对战**（PvP） |
| AI 难度 - 简单 | 随机落子 + 仅阻止五连 |
| AI 难度 - 一般 | 贪心评分，综合进攻与防守 |
| AI 难度 - 困难 | Minimax + Alpha-Beta 剪枝，搜索深度 4 |
| 悔棋 | PvC 模式撤销人类+AI各一步；PvP 撤销一步 |
| 落子预览 | 鼠标悬停显示半透明棋子预览 |
| 触屏支持 | 支持手机/平板触摸落子 |

## 游戏规则

- 标准 15×15 棋盘，黑棋先行
- 任意一方在横/纵/斜方向连成**五子**即获胜
- 棋盘落满无胜负则判**平局**

## AI 算法说明

### 简单难度
- 能赢则赢（检测五连）
- 阻止对手五连
- 否则随机选取相邻空位

### 一般难度
使用评分函数对所有候选位置打分：
- 进攻分 + 1.2 × 防守分
- 根据连子数量和两端开放情况赋予不同权重（活四 > 冲四 > 活三 > ...）

### 困难难度
- Minimax 博弈树搜索，深度 4
- Alpha-Beta 剪枝大幅减少计算量
- 候选位置限定在已有棋子附近 2 格范围内
- 先进行快速赢局/阻挡检测，再启动深度搜索
